# 代码片段管理

`~/.claude/snippets/` 目录作为个人代码片段库。两套模式覆盖不同需求：

- **代码片段**：存实际代码，下次写同类功能时调用适配
- **效果浓缩**：只存行为规则，跨语言跨平台还原效果

---

# 一、代码片段

## 格式

每个片段一个 `.md` 文件，YAML frontmatter + 代码体：

```
---
name: {用户命名}
description: {一句话：做什么、适用场景}
tags: [{标签}]
language: {语言}
created: {YYYY-MM-DD}
---

{代码}
```

## 保存

触发：**保存这段代码为 / 记住这个 / 收藏 / save as / snippet this**

1. 识别对话中最近编写的代码
2. 创建 `~/.claude/snippets/{name}.md`，用上述格式
3. 去掉项目特定硬编码（路径、密钥、配置值）
4. 同名片段先询问是否覆盖

## 检索

触发：**我要XX功能 / 调出XX / 用之前的XX / I need XX**

1. `ls ~/.claude/snippets/` 列出所有片段
2. 按文件名、name、description、tags 模糊匹配
3. 读最匹配的文件
4. **适配当前项目后补全**（语言语法、导入路径、命名风格、框架 API）

## 浏览与删除

浏览：**我有哪些片段 / 列出片段 / list snippets** → 读所有 frontmatter，输出 name + description + tags
删除：**删除XX片段 / delete snippet XX** → 删除对应文件

# 二、收工回顾（效果浓缩）

触发：**收工 / 下班 / 今日收尾 / 项目收尾 / 结束项目 / 回顾代码**

目标：扫描项目改动，识别有跨场景价值的交互效果/算法/数据模式，浓缩为**纯行为规则**存入片段库。不存代码，只存"实现说明书"——让未来在任何语言任何平台都能还原同样效果。

## 流程

### 第一步：确定扫描范围

- 含"今日""下班"等词 → 扫描今天的所有 commit 改动
- 含"项目""收尾" → 扫描整个项目 diff vs main/master
- 含"回顾代码" → 询问用户指定范围（目录/时间段/commit 范围）

### 第二步：识别候选模块

按以下标准筛选：

| 值得浓缩 | 跳过 |
|----------|------|
| 交互效果（拖拽、滑动、手势、过渡动画） | 纯业务逻辑（订单处理、用户认证） |
| 不常见的算法/公式组合 | 框架样板代码（路由配置、脚手架文件） |
| 数据结构设计（非直觉的组织方式） | 配置文件、环境变量 |
| 状态机/复杂流程控制 | 能一眼看懂的三行函数 |
| 踩坑后的正确实现（换了平台容易忘记） | CSS 装饰性样式（颜色、圆角、阴影） |

### 第三步：输出候选列表

以表格形式呈现，让用户勾选：

```
📋 本次可浓缩的代码模块：

| # | 模块 | 效果要点 | 文件来源 |
|----|------|----------|----------|
| 1 | 全页滑动切换 | 多面板横向滑动，当前居中吸附 | Carousel.tsx |
| 2 | 拖拽排序 | 原地占位 + 跟随指针 + 插入动画 | DragList.tsx |
| 3 | 虚拟滚动 | 仅渲染可视区 ± 缓冲区 item | VirtualList.tsx |

回复编号（如 1,3）选择要浓缩的模块，"全要"或"算了"。
```

### 第四步：写入浓缩片段

对每个确认的模块，生成浓缩片段文件 `~/.claude/snippets/{name}.md`。

同名片段询问：覆盖 / 合并（追加内容）/ 跳过。

## 浓缩片段格式

```yaml
---
name: {功能名}
essence: {一句话——在任何平台复现此效果需要知道的最少事实}
key_techniques: [{核心技术点}]
type: effect
created: {YYYY-MM-DD}
source: {来源项目，可选}
---

## {维度1，如状态机}
{状态} → {状态} ({触发条件})
...

## {维度2，如关键行为}
{阶段}: {输入→输出的规则}
...

## {维度3，如参数}
{参数名} = {值或公式}
...

## 禁止项（可选）
- {换了平台容易忘记的坑}
```

**维度标题按需命名**，不是死格式。常见的：

- `状态机` — 有显式状态转换的交互
- `关键行为` — 每个阶段的输入→输出规则
- `参数` — 所有可调数值和比例关系
- `数据模型` — 非直觉的数据结构设计
- `算法核心` — 公式/流程/边界条件
- `禁止项` — 必须避免的行为

**内容原则**：三不存
- 不存 import/export
- 不存框架 API 调用（CSS 属性、React Hook 名等——除非 API 本身是精髓）
- 不存类型声明

只存"换个平台我要对什么值做什么操作"，换平台后自然映射到对应 API。

**例——全页滑动切换的浓缩片段：**

```yaml
---
name: 全页滑动切换
essence: 多面板横向滑动，当前面板居中吸附，其余自动对齐边缘
key_techniques: [三阶段状态机, 惯性跟踪, 阈值吸附, rubber-band 回弹]
type: effect
created: 2026-05-07
---

## 状态机
idle → tracking (pointerdown)
tracking → settling (pointerup)
settling → idle (动画结束)

## 关键行为
跟踪阶段:
  面板位置 = 基准偏移 + 本次位移
  基准偏移 = 上次 settle 的目标位置
  超出边界: 位移 × 0.3 (rubber-band)

沉降阶段 (pointerup):
  若 |本次位移| > 面板宽 × 0.3 → 切到相邻面板
  否则 → 回弹到当前面板
  动画: ease-out, 时长 = min(面板宽/3, 300) ms

## 参数
面板宽 = 容器宽 × 0.85
间距 = 16px
面板 n 的静态位置 = n × (面板宽 + 间距)
切换阈值 = 面板宽 × 0.3
rubber-band 系数 = 0.3
沉降时长上限 = 300ms

## 禁止项
- tracking 中禁止沉降动画（需 cancelAnimationFrame）
- settling 中忽略所有 pointer 事件
```

# 三、红线

| 禁止 | 必须 |
|------|------|
| 代码片段检索后原样粘贴 | 根据当前项目语言/框架/风格适配 |
| 浓缩片段要求写成具体实现代码 | 只写行为规则，语言无关 |
| 保存硬编码的项目特定值 | 只保留通用逻辑或参数化公式 |
| 不读文件就猜测内容 | 先读文件确认 |
| 收工回顾时列出所有文件变更 | 只列有跨场景复用价值的模块 |
| 浓缩片段内写 import/export/JSX/类型声明 | 只留行为规则和关键算法 |
